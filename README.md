# Nootropic Stack Builder

A single-page interactive tool for designing, visualising, and sharing nootropic stacks. Zero dependencies — pure HTML, CSS, and JavaScript.

**Live demo:** https://nootroblog.com/stack-builder/

## Features

- **45+ compounds** — racetams, adaptogens, choline sources, amino acids, herbs, vitamins, peptides
- **Interaction detection** — synergy flags (✓) and caution warnings (⚠) for established compound interactions
- **Stack summary** — pills/day, estimated monthly cost, benefit coverage bars, balance score
- **URL sharing** — stack state encoded in `?s=` parameter; share any stack with a link
- **Embed generator** — iframe snippet with your current stack pre-loaded and `?embed=1` mode (hides nav/hero)
- **Search + category filter** — quickly find compounds by name or category
- **Hover tooltips** — mechanism, half-life, cost estimate on hover
- **Dark mode** — dark-first design, no light mode toggle needed

## Deploy on Netlify

1. **Drag & drop:** Go to [netlify.com/drop](https://app.netlify.com/drop), drag the `stack-builder/` folder onto the page. Done.

2. **Or via Git:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR/repo.git
   git push -u origin main
   ```
   Then connect the repo in the Netlify dashboard. No build command needed — publish directory is `.` (root).

3. **Custom domain:** In Netlify → Domain settings → Add custom domain.

## Deploy on Vercel

1. Install Vercel CLI (optional): `npm i -g vercel`
2. Run `vercel` in the `stack-builder/` directory and follow prompts.
3. Or connect your GitHub repo in the [Vercel dashboard](https://vercel.com/new). Framework preset: **Other**. No build command. Output directory: `.`

## Deploy on Cloudflare Pages

1. Push to GitHub.
2. In Cloudflare dashboard → Pages → Create a project → Connect to Git.
3. Build command: *(leave blank)*. Build output directory: `/`.
4. Deploy.

Or use Wrangler:
```bash
npx wrangler pages deploy . --project-name stack-builder
```

## Updating the database

All compound data is in `data.js`. Each entry follows this shape:

```js
{
  id: 'compound-id',          // kebab-case, used in URL params
  name: 'Display Name',
  category: 'racetam',        // racetam|choline|adaptogen|amino|herb|vitamin|peptide|other
  benefits: ['focus', 'memory'], // from: focus memory mood sleep anxiety energy neuroprotection
  dose: '400 mg × 2/day',
  halfLife: '5–6 h',
  mechanism: 'Short plain-English explanation of how it works.',
  costRange: [20, 40],        // [min, max] USD per month
  pillsPerDay: 2,
  notes: 'Optional extra note shown in tooltip.',
}
```

Interactions are in the `INTERACTIONS` array:

```js
{
  compounds: ['compound-a', 'compound-b'],
  type: 'synergy',   // or 'caution'
  note: 'Explanation of the interaction.',
}
```

Only include interactions with established pharmacological mechanisms.

## Embedding

The embed snippet generated by the tool looks like:

```html
<iframe
  src="https://nootroblog.com/stack-builder/?s=piracetam,alpha-gpc&embed=1"
  width="100%"
  height="700"
  frameborder="0"
  style="border-radius:12px;border:1px solid #2a2a2a;"
  title="Nootropic Stack Builder — Powered by NootroBlog"
></iframe>
```

The `?embed=1` parameter hides the nav and hero, leaving just the two-column builder.
